    Why taskdock

    Small on purpose. Yours entirely.

    Every choice bends toward one thing: a task list that stays out of your way and never leaves your machine.

    01

    Your data, one file

    Everything lives in a local SQLite file. No servers to trust, no account to create, nothing phoning home. Back it up by copying one file.

    02

    Persists through anything

    Close the tab, reboot the laptop, kill the server — your todos are still there when you come back. Persistence is the whole point.

    03

    A real REST backend

    Not just a UI. Five clean endpoints mean you can script it, curl it, or build your own front end on top.

    04

    Instant, no bloat

    Plain HTML and JavaScript over a lightweight Express server. It opens in your browser and just works — no build step to run the app.

    05

    Hackable in an afternoon

    A tiny, readable TypeScript codebase split into data, routes, and UI. Add a field or a filter without spelunking through a framework.

    06

    Zero native headaches

    Storage uses Node's built-in SQLite — nothing to compile, no fragile native modules. Clone, install, run.

    The API

    A backend you can talk to.

    Everything the UI does goes through these endpoints. So can you — from a script, a shortcut, or your own client.

    GET /api/todosList every todo, newest first
    POST /api/todosCreate a todo  ·  { "title": "…" }
    GET /api/todos/:idFetch a single todo
    PATCH /api/todos/:idUpdate  ·  { "title"?, "done"? }
    DELETE /api/todos/:idRemove a todo
